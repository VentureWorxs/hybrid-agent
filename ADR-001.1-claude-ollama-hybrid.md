## 1. Context

ADR-001 established a working **Claude Code + Ollama (Qwen3-235B-A22B) hybrid** via a stdio MCP server. That foundation handles **delegation** (Claude routes bulk/sensitive tasks to local Qwen3) but has gaps:

- **No persistent memory across sessions.** Every Claude Code session starts cold — prior routing decisions, trust scores, and approvals are lost.
- **No audit trail.** Decisions, agent invocations, and boundary enforcement are not recorded. Compliance evidence is impossible to produce after the fact.
- **No cost/performance metrics.** Token savings from Ollama routing are invisible.
- **No multi-machine coordination.** A decision made on the M3 Mac is unknown to the Windows workstation.
- **No HIPAA evidence trail.** For Propel work, there is no proof that PHI stayed local.

The Propel context is particularly important: HIPAA requires documented evidence of data handling controls. ADR-001 *enforces* PHI confinement (by using Ollama), but does not *prove* it after the fact.

Additionally, the broader NicheWorxs portfolio (BrushWorxs, OriginWorxs, VentureWorxs, etc.) would benefit from a unified audit dashboard — viewable on venworxs.com — showing cost savings, routing decisions, and compliance posture across all brands.

ADR-001.1 layers orchestration, decision caching, and dual-backend audit (local SQLite + Cloudflare D1 replica) on top of the existing MCP bridge.

## 2. Decision

Layer **three new components** above the existing MCP server, all backed by a unified **append-only event log**:

### Layer 1 — Audit Logger (foundational)

- **Append-only event log** in local SQLite (`audit_events` table)
- **Hash-chain integrity** — each event references the SHA-256 of the prior event (tamper-evident)
- **Hybrid schema** — typed columns for queryable fields; JSON column for flexible details
- **Multi-tenant** — every event tagged with `tenant_id` (`sam-personal`, `nicheworxs`, `propel`)
- **Machine/session identity** — every event tagged with `machine_id`, `session_id`, `agent_version`

### Layer 2 — Decision Cache (derived from audit log)

- **Materialized view** of decisions, rebuilt by replaying `decision_made` events
- **Cross-session persistence** — decisions survive Claude Code restarts
- **Trust-based scope escalation** — repeated successful decisions auto-elevate scope
- **TTL-based expiry** — sensitive decisions expire automatically

### Layer 3 — Orchestrator (consumes both)

- **Task classification** — complexity (0-10) + sensitivity (`public` / `internal` / `confidential` / `sensitive_phi`)
- **Intelligent routing** — Ollama for simple/sensitive; Claude for complex reasoning; specialized agents for domain work
- **DAG-aware execution** — parallel where dependencies allow
- **Decision cache lookup** — reuses cached routing decisions to skip redundant approvals
- **Compliance boundary enforcement** — PHI tasks blocked from routing to Claude

### Dual-Backend Storage

- **Local SQLite** — source of truth, always available, full fidelity
- **Cloudflare D1** — replica for dashboard access, sanitized (no raw PHI)
- **Batch sync every 5 minutes** — offline-resilient queue with retry
- **Storage abstraction** — same `AuditStorage` interface for both backends

### Future Layer 4 — Audit Dashboard

Out of scope for this ADR (separate work). Architecture preview in Section 14.

## 3. Consequences

### Positive

- **Compliance evidence** — every decision, agent invocation, and boundary enforcement is immutably logged with hash-chain verification
- **Persistent memory** — Claude Code sessions are no longer cold; prior routing decisions and trust scores carry forward
- **Cost visibility** — token savings from Ollama routing are tracked per task, per tenant, per day
- **HIPAA proof** — audit trail demonstrates PHI never left the local boundary (provable, not just policy)
- **Multi-machine awareness** — decisions on M3 Mac sync to D1; Windows workstation sees them on next sync
- **Dashboard-ready** — D1 replica is queryable from any web context (venworxs.com or internal Propel tool)
- **Tamper evidence** — hash-chain integrity check detects any modification to historical events
- **Replay capability** — entire state (decision cache, trust scores, analytics) can be rebuilt from event log

### Negative / Risks

- **Increased complexity** — three new modules + storage layer + sync worker vs. simple stdio MCP
- **Write amplification** — every agent invocation generates 1-3 audit events (estimate: ~10 KB per event in SQLite)
- **Sync failure modes** — network outages queue events locally; queue can grow if outage is long
- **Schema migrations** — must be coordinated across local SQLite and D1 (forward-only)
- **Hash-chain rigidity** — fixing a buggy event requires emitting a *correction* event (immutability is the point)
- **D1 quota** — Cloudflare D1 free tier: 5 GB storage, 25M reads/day, 50K writes/day — adequate for personal use but worth monitoring

### Neutral

- The existing MCP server (`mcp_ollama_server.py`) is unchanged in behavior — it just gains an audit hook wrapper
- Claude Code configuration is unchanged at the `settings.json` level
- Ollama installation, model, and tools are unchanged

**Data flow for a delegated task with audit:**

1. User prompt → Claude Code
2. Claude classifies task complexity + sensitivity
3. Claude calls `get_decision` MCP tool → checks cache for prior routing decision
4. Claude routes: Ollama (local) or stays in Claude (cloud)
5. If Ollama: stdio → MCP server → `localhost:11434` → Qwen3 → response
6. Claude calls `audit_log_event` MCP tool → event appended to local SQLite with hash chain
7. Claude calls `cache_decision` (if new decision) → cache materialized view updated
8. Sync worker (background, every 5 min) reads unsynced events
9. Sanitization pipeline scrubs PHI/PII based on `sensitivity_level`
10. Sanitized batch POSTed to Cloudflare Worker (auth via Cloudflare Access)
11. Worker writes to D1; returns success/failure per event
12. Sync worker marks events synced locally; retries failures with exponential backoff

## 7. Schema Design

### 7.1 Append-Only Event Log

The **`audit_events`** table is the single source of truth. All other state (decision cache, analytics) is derived from it.

```sql
-- schema/sqlite_schema.sql
-- Compatible with both SQLite (3.38+) and Cloudflare D1

PRAGMA journal_mode = WAL;
PRAGMA foreign_keys = ON;

-- Schema version tracking
CREATE TABLE IF NOT EXISTS schema_version (
    version INTEGER PRIMARY KEY,
    applied_at TEXT NOT NULL,
    description TEXT
);

-- Core append-only audit log
CREATE TABLE IF NOT EXISTS audit_events (
    -- Identity (immutable, used in hash)
    event_id          TEXT PRIMARY KEY,            -- ULID (sortable UUID)
    sequence_number   INTEGER NOT NULL,            -- Monotonic per (machine_id)
    
    -- Hash chain (tamper evidence)
    previous_hash     TEXT,                        -- SHA-256 of prior event's event_hash
    event_hash        TEXT NOT NULL,               -- SHA-256 of this event's canonical form
    
    -- Multi-tenant + machine/session identity
    tenant_id         TEXT NOT NULL,               -- 'sam-personal' | 'nicheworxs' | 'propel'
    machine_id        TEXT NOT NULL,               -- UUID per physical machine
    session_id        TEXT NOT NULL,               -- UUID per Claude Code session
    agent_version     TEXT NOT NULL,               -- semver of agent at write time
    
    -- Temporal
    timestamp         TEXT NOT NULL,               -- ISO8601 with timezone
    
    -- Event classification (queryable, indexed)
    event_type        TEXT NOT NULL,               -- See event_type enum below
    actor             TEXT NOT NULL,               -- 'user' | 'orchestrator' | 'claude' | 'ollama-local' | 'agent:{domain}'
    subject_type      TEXT,                        -- 'task' | 'subtask' | 'decision' | 'system'
    subject_id        TEXT,                        -- ID of the thing being acted upon
    action            TEXT NOT NULL,               -- Short human-readable verb phrase
    
    -- Scope & approval
    scope_level       INTEGER,                     -- 0=minimal, 1=standard, 2=elevated, 3=full
    approval_status   TEXT,                        -- 'auto' | 'approved' | 'rejected' | 'pending'
    approval_by       TEXT,                        -- User or 'system'
    
    -- Sensitivity (drives sanitization)
    sensitivity_level TEXT NOT NULL DEFAULT 'public', -- 'public' | 'internal' | 'confidential' | 'sensitive_phi'
    
    -- Cost & performance metrics
    tokens_used       INTEGER DEFAULT 0,
    cost_usd          REAL DEFAULT 0.0,
    execution_time_ms INTEGER,
    
    -- Routing decision tracking
    agent_routed_to   TEXT,                        -- 'ollama-local' | 'claude-api' | 'agent:{domain}'
    
    -- Boundary enforcement
    boundary_enforced INTEGER DEFAULT 0,           -- 0/1 boolean
    
    -- Hybrid: flexible details
    details           TEXT,                        -- JSON blob — queryable via json_extract()
    
    -- Sync state (local-only fields; not synced)
    synced_to_d1      INTEGER DEFAULT 0,           -- 0/1
    synced_at         TEXT,
    sync_attempts     INTEGER DEFAULT 0,
    sync_error        TEXT,
    
    -- Uniqueness: no duplicate sequence per machine
    UNIQUE(machine_id, sequence_number)
);

-- Indexes for common query patterns
CREATE INDEX IF NOT EXISTS idx_events_tenant_time     ON audit_events(tenant_id, timestamp);
CREATE INDEX IF NOT EXISTS idx_events_type            ON audit_events(event_type);
CREATE INDEX IF NOT EXISTS idx_events_subject         ON audit_events(subject_type, subject_id);
CREATE INDEX IF NOT EXISTS idx_events_sync            ON audit_events(synced_to_d1, timestamp);
CREATE INDEX IF NOT EXISTS idx_events_sensitivity     ON audit_events(sensitivity_level);
CREATE INDEX IF NOT EXISTS idx_events_session         ON audit_events(session_id);
CREATE INDEX IF NOT EXISTS idx_events_machine_seq     ON audit_events(machine_id, sequence_number);

-- Tenant registry
CREATE TABLE IF NOT EXISTS tenants (
    tenant_id    TEXT PRIMARY KEY,
    display_name TEXT NOT NULL,
    created_at   TEXT NOT NULL,
    metadata     TEXT                              -- JSON: compliance flags, retention policy, etc.
);

INSERT OR IGNORE INTO tenants (tenant_id, display_name, created_at, metadata) VALUES
    ('sam-personal', 'Sam (Personal)',   datetime('now'), '{"phi_allowed": false}'),
    ('nicheworxs',   'NicheWorxs',       datetime('now'), '{"phi_allowed": false}'),
    ('propel',       'Propel (HIPAA)',   datetime('now'), '{"phi_allowed": true, "retention_days": 2555}');

-- Machine registry
CREATE TABLE IF NOT EXISTS machines (
    machine_id   TEXT PRIMARY KEY,
    hostname     TEXT,
    platform     TEXT,                             -- 'darwin-arm64' | 'win32-x64' | etc.
    first_seen   TEXT NOT NULL,
    last_seen    TEXT NOT NULL,
    metadata     TEXT                              -- JSON
);
```

```

- 'task_completed'            — Task finished successfully
- 'task_failed'               — Task aborted with error
- 'subtask_started'
- 'subtask_completed'
- 'subtask_failed'
```

### 7.2 Materialized Views (Derived State)

The **decision cache** is rebuilt from `decision_made` and `decision_reused` events. Rather than a SQL VIEW (which would recompute every query), we use a **physical table** updated by application code or a trigger.

```sql
-- Materialized view: latest cached decisions
CREATE TABLE IF NOT EXISTS decision_cache (
    decision_id       TEXT PRIMARY KEY,            -- Maps to source event_id
    tenant_id         TEXT NOT NULL,
    context_hash      TEXT NOT NULL,               -- SHA-256 of sanitized context
    decision_type     TEXT NOT NULL,               -- 'routing' | 'permission' | 'scope_level' | etc.
    decision_value    TEXT NOT NULL,               -- JSON-encoded decision
    routed_to         TEXT,                        -- 'ollama-local' | 'claude-api' | 'agent:{domain}'
    scope_level       INTEGER NOT NULL,
    trust_score       REAL NOT NULL DEFAULT 0.5,   -- 0.0-1.0
    reuse_count       INTEGER NOT NULL DEFAULT 0,
    tokens_saved      INTEGER DEFAULT 0,
    cost_saved        REAL DEFAULT 0.0,
    created_at        TEXT NOT NULL,
    last_reused_at    TEXT,
    expires_at        TEXT,                        -- TTL for sensitive decisions
    metadata          TEXT,                        -- JSON
    
    UNIQUE(tenant_id, context_hash, decision_type)
);

CREATE INDEX IF NOT EXISTS idx_cache_tenant_context  ON decision_cache(tenant_id, context_hash);
CREATE INDEX IF NOT EXISTS idx_cache_expires         ON decision_cache(expires_at);
CREATE INDEX IF NOT EXISTS idx_cache_trust           ON decision_cache(trust_score);
```

**Rebuild logic** (event sourcing):

```python
def rebuild_decision_cache(storage: AuditStorage) -> None:
    """Replay all decision_made and decision_reused events to rebuild cache."""
    storage.execute("DELETE FROM decision_cache")
    
    for event in storage.iter_events(event_type='decision_made', order='asc'):
        upsert_cache_entry(event)
    
    for event in storage.iter_events(event_type='decision_reused', order='asc'):
        increment_reuse_count(event)
```

**Refresh logic** runs after each sync batch:

```python
def refresh_analytics(storage: AuditStorage, since: datetime) -> None:
    """Incrementally update analytics tables from new events since `since`."""
    storage.execute("""
        INSERT OR REPLACE INTO analytics_daily_cost
        SELECT tenant_id, date(timestamp) as date, agent_routed_to,
               COUNT(*), SUM(tokens_used), SUM(cost_usd), AVG(execution_time_ms),
               datetime('now')
        FROM audit_events
        WHERE event_type = 'agent_invoked' AND timestamp >= ?
        GROUP BY tenant_id, date(timestamp), agent_routed_to
```

### 7.4 Migration Strategy

**Approach**: Forward-only numbered SQL files. Each migration runs once per database, tracked in `schema_version`.

```
schema/migrations/
├── 001_initial.sql              ← Base schema (audit_events, tenants, machines)
├── 002_decision_cache.sql       ← Decision cache materialized view
├── 003_analytics_views.sql      ← Analytics tables
├── 004_indexes_phase2.sql       ← Performance indexes
└── ...
```

**Migration runner** (Python):

```python
def apply_migrations(conn, migrations_dir: Path) -> None:
    current = conn.execute("SELECT MAX(version) FROM schema_version").fetchone()[0] or 0
    for sql_file in sorted(migrations_dir.glob("*.sql")):
        version = int(sql_file.stem.split("_")[0])
        if version <= current:
            continue
        with conn:
            conn.executescript(sql_file.read_text())
            conn.execute(
                "INSERT INTO schema_version VALUES (?, ?, ?)",
                (version, datetime.utcnow().isoformat(), sql_file.stem)
            )
        print(f"Applied migration {sql_file.name}")
```

**D1 migration** uses `wrangler d1 migrations apply`:

| Feature | SQLite | D1 | Mitigation |
|---------|--------|----|--------|
| JSON1 functions | ✅ | ✅ | Use `json_extract()` consistently |

| Triggers | ✅ | ✅ | Use sparingly; prefer application logic |

## 8. Storage Abstraction Layer

Both SQLite and D1 implement the same interface, so application code is backend-agnostic.

```python
# audit/storage_abstraction.py
from abc import ABC, abstractmethod
from typing import Iterator, Optional
from .models import AuditEvent

class AuditStorage(ABC):
    """
    Backend-agnostic interface for audit event storage.
    
    Implementations:
      - SQLiteAuditStorage: local source of truth
      - D1AuditStorage: cloud replica (read-mostly from agent's perspective)
    """
    
    @abstractmethod
    def append_event(self, event: AuditEvent) -> str:
        """
        Append a new event to the log. Computes hash chain.
        Returns the event_id.
        Raises HashChainError if previous_hash doesn't match.
        """
        ...
    
    @abstractmethod
    def get_event(self, event_id: str) -> Optional[AuditEvent]:
        ...
    
    @abstractmethod
    def iter_events(
        self,
        tenant_id: Optional[str] = None,
        event_type: Optional[str] = None,
        since: Optional[str] = None,
        until: Optional[str] = None,
        order: str = 'asc',
        limit: Optional[int] = None,
    ) -> Iterator[AuditEvent]:
        ...
    
    @abstractmethod
    def get_latest_hash(self, machine_id: str) -> Optional[str]:
        """Return event_hash of most recent event for this machine (for chaining)."""
        ...
    
    @abstractmethod
    def get_latest_sequence(self, machine_id: str) -> int:
        """Return highest sequence_number for this machine, or 0 if none."""
        ...
    
    @abstractmethod
    def get_unsynced(self, limit: int = 100) -> list[AuditEvent]:
        """Return events not yet synced to remote. SQLite only — D1 returns []."""
        ...
    
    @abstractmethod
    def mark_synced(self, event_ids: list[str], synced_at: str) -> None:
        """Mark events as successfully synced to remote."""
        ...
    
    @abstractmethod
    def mark_sync_failed(self, event_id: str, error: str) -> None:
        """Increment sync_attempts and record error."""
        ...
    
    @abstractmethod
    def verify_chain(
        self, 
        machine_id: str, 
        start_seq: Optional[int] = None, 
        end_seq: Optional[int] = None
    ) -> tuple[bool, Optional[str]]:
        """
        Walk the hash chain for the given machine. 
        Returns (is_valid, error_message_if_invalid).
        """
        ...
```

## 9. Sync Strategy (Local → D1)

### 9.1 Sync Loop

A background process runs every 5 minutes (configurable via `SYNC_INTERVAL_SECONDS`):

```python
# audit/sync_worker.py
import time, logging
from datetime import datetime
from .sqlite_storage import SQLiteAuditStorage
from .sanitization import SanitizationPipeline
from .d1_client import D1WorkerClient

def sync_loop(storage: SQLiteAuditStorage, sanitizer: SanitizationPipeline, 
              d1_client: D1WorkerClient, batch_size: int = 100, 
              interval_seconds: int = 300) -> None:
    while True:
        try:
            unsynced = storage.get_unsynced(limit=batch_size)
            if not unsynced:
                time.sleep(interval_seconds)
                continue
            
            # Sanitize events (drops PHI, redacts PII)
            sanitized = [sanitizer.sanitize(ev) for ev in unsynced]
            sanitized = [ev for ev in sanitized if ev is not None]
            
            # Push batch to Cloudflare Worker
            response = d1_client.push_batch(sanitized)
            
            # Mark successes; record failures
            now = datetime.utcnow().isoformat()
            for event_id, status in response.items():
                if status == 'success':
                    storage.mark_synced([event_id], now)
                else:
                    storage.mark_sync_failed(event_id, status)
            
            logging.info(f"Synced {len(sanitized)} events, {len(unsynced) - len(sanitized)} dropped (PHI), {sum(1 for s in response.values() if s != 'success')} failed")
        
        except Exception as e:
            logging.exception("Sync loop error")
            time.sleep(interval_seconds)
        
        time.sleep(interval_seconds)
```

**Retry logic**:
- Failed events stay in `synced_to_d1 = 0` state
- Next sync run retries them
- After 5 attempts, log warning but keep retrying (no permanent failure state)
- Exponential backoff on transport errors (network down)

**Starting the sync worker**:

```bash
# Run as a long-lived process (macOS: launchd, Windows: Task Scheduler, or just a terminal)
python -m audit.sync_worker --tenant=sam-personal --interval=300

# Or as a systemd service on Linux, launchd plist on macOS, etc.
```

### 9.2 Conflict Resolution

D1 has `UNIQUE(event_id)` constraint. If a duplicate event arrives (e.g., due to a retry race):

- Worker uses `INSERT OR IGNORE` and returns `duplicate_skipped` status
- Sync worker treats `duplicate_skipped` as success (marks synced)

### 9.3 Out-of-Order Sync

Events are written to D1 in batches. Within a batch, sequence numbers may not be contiguous (if events were dropped by sanitization). D1 stores what it gets; the hash chain is verified against **local SQLite** which always has the full ordered chain.

**Decision**: Hash chain integrity is verified locally, not on D1. D1 is a derived replica; it doesn't claim to be tamper-evident.

## 10. Sanitization Pipeline

Before any event leaves the local machine for D1, it passes through sanitization rules based on `sensitivity_level`.

```python
# audit/sanitization.py
from typing import Optional
from .models import AuditEvent

class SanitizationPipeline:
    """
    Sanitization rules by sensitivity_level:
    
    - 'public':           No changes. Sync as-is.
    - 'internal':         Redact PII fields (emails, phone numbers, names in details).
    - 'confidential':     Drop raw content. Keep metadata (event_type, timestamp, cost, 
                          agent_routed_to). Replace details with a summary.
    - 'sensitive_phi':    DROP the event entirely. Emit a metadata-only "shadow" event 
                          proving it happened (event_type, timestamp, machine_id, 
                          tenant_id, agent_routed_to) — for HIPAA evidence — but with 
                          NO content references and NO subject_id.
    """
    
    def sanitize(self, event: AuditEvent) -> Optional[AuditEvent]:
        level = event.sensitivity_level
        
        if level == 'public':
            return event
        
        if level == 'internal':
            return self._redact_pii(event)
        
        if level == 'confidential':
            return self._strip_to_metadata(event)
        
        if level == 'sensitive_phi':
            return self._emit_shadow_event(event)
        
        # Unknown level — fail closed
        return None
    
    def _redact_pii(self, event: AuditEvent) -> AuditEvent:
        # Apply regex/heuristic redaction on string fields in details
        # ...
        return event
    
    def _strip_to_metadata(self, event: AuditEvent) -> AuditEvent:
        # Keep typed columns; replace details with summary
        event.details = json.dumps({
            "sanitized": True,
            "original_details_size_bytes": len(event.details or ""),
            "summary": f"{event.event_type} event redacted at sync time"
        })
        return event
    
    def _emit_shadow_event(self, event: AuditEvent) -> AuditEvent:
        # HIPAA pattern: prove the event happened, reveal nothing about its content
        shadow = AuditEvent(
            event_id=event.event_id,                # Same ID for traceability
            sequence_number=event.sequence_number,
            previous_hash=event.previous_hash,
            event_hash=event.event_hash,            # Hash of *original* event
            tenant_id=event.tenant_id,
            machine_id=event.machine_id,
            session_id=event.session_id,
            agent_version=event.agent_version,
            timestamp=event.timestamp,
            event_type=event.event_type,
            actor=event.actor,
            subject_type=None,                      # REDACTED
            subject_id=None,                        # REDACTED
            action="[PHI event — confined to local storage per HIPAA]",
            scope_level=event.scope_level,
            approval_status=event.approval_status,
            approval_by=event.approval_by,
            sensitivity_level='sensitive_phi',
            tokens_used=event.tokens_used,          # Aggregate metric OK
            cost_usd=event.cost_usd,
            execution_time_ms=event.execution_time_ms,
            agent_routed_to=event.agent_routed_to,  # Must be 'ollama-local' for PHI
            boundary_enforced=1,                    # Boundary enforced by definition
            details=json.dumps({
                "phi_confinement": "enforced",
                "compliance_framework": "HIPAA"
            })
        )
        return shadow
```

**Key property**: A PHI event in D1 proves *it happened* (HIPAA needs that), but reveals *nothing about the data* (HIPAA requires that).

## 11. Hash-Chain Integrity

Each event includes `previous_hash` and `event_hash`. Tampering with any historical event invalidates all subsequent hashes.

```python
# audit/hash_chain.py
import hashlib, json

# Fields included in the hash (immutable, canonical)
HASH_FIELDS = [
    'event_id', 'sequence_number', 'previous_hash',
    'tenant_id', 'machine_id', 'session_id', 'agent_version',
    'timestamp', 'event_type', 'actor',
    'subject_type', 'subject_id', 'action',
    'scope_level', 'approval_status', 'approval_by',
    'sensitivity_level',
    'tokens_used', 'cost_usd', 'execution_time_ms',
    'agent_routed_to', 'boundary_enforced',
    'details',
]

def compute_event_hash(event, previous_hash: str = None) -> str:
    """
    Compute SHA-256 of canonical event representation.
    Fields are sorted and serialized via JSON with sort_keys=True for stability.
    """
    canonical = {f: getattr(event, f) for f in HASH_FIELDS}
    canonical['previous_hash'] = previous_hash or ''
    serialized = json.dumps(canonical, sort_keys=True, separators=(',', ':'), default=str)
    return hashlib.sha256(serialized.encode('utf-8')).hexdigest()


def verify_chain(events: list) -> tuple[bool, str]:
    """
    Walk the chain in order. Returns (is_valid, error_message).
    """
    expected_prev = None
    expected_seq = 0
    
    for ev in events:
        # Sequence must be monotonic per machine
        if ev.sequence_number != expected_seq + 1:
            return False, f"Sequence gap at event {ev.event_id}: expected {expected_seq + 1}, got {ev.sequence_number}"
        
        # previous_hash must match the prior event's event_hash
        if ev.previous_hash != expected_prev:
            return False, f"Chain break at event {ev.event_id}: previous_hash mismatch"
        
        # event_hash must match recomputed value
        recomputed = compute_event_hash(ev, ev.previous_hash)
        if recomputed != ev.event_hash:
            return False, f"Hash mismatch at event {ev.event_id}: stored != recomputed"
        
        expected_prev = ev.event_hash
        expected_seq = ev.sequence_number
    
    return True, None
```

**When to verify**:
- On startup of sync worker (sanity check)
- On demand via `verify_audit_chain` MCP tool
- Before generating compliance reports
- After any unexpected database restart

**Tamper detection**: If `verify_chain` returns False, the integrity break is logged as a `system_error` audit event (which itself becomes part of the chain going forward). Sam should investigate manually.

## 12. Event Sourcing & State Reconstruction

The `decision_cache` table and all `analytics_*` tables are **derived state**. They can be deleted and rebuilt from `audit_events` at any time.

```python
# audit/event_sourcing.py
def rebuild_all_derived_state(storage: SQLiteAuditStorage) -> None:
    """
    Drop and rebuild all derived tables from the audit_events log.
    Use cases:
      - After schema migration that changes derived state
      - After detecting corruption
      - For testing / debugging
    """
    with storage.transaction():
        storage.execute("DELETE FROM decision_cache")
        storage.execute("DELETE FROM analytics_daily_cost")
        storage.execute("DELETE FROM analytics_ollama_savings")
        storage.execute("DELETE FROM analytics_compliance")
        storage.execute("DELETE FROM analytics_task_frequency")
        
        # Replay decision cache
        for event in storage.iter_events(event_type='decision_made', order='asc'):
            apply_decision_made(storage, event)
        for event in storage.iter_events(event_type='decision_reused', order='asc'):
            apply_decision_reused(storage, event)
        
        # Rebuild analytics
        refresh_analytics_all(storage)
    
    logging.info("Derived state rebuilt from audit_events")
```

**Why this matters**:
- **Debugging**: Replay events on a copy of the database to reproduce a bug
- **Schema evolution**: Add a new analytics view, rebuild from scratch — no migration of derived data needed
- **Recovery**: Corrupted derived state? Rebuild. Audit log is the only thing that must be backed up.

```python
# orchestrator/decision_cache.py
import hashlib, json
from datetime import datetime, timedelta
from typing import Optional
from audit.sqlite_storage import SQLiteAuditStorage

class DecisionCache:
    def __init__(self, storage: SQLiteAuditStorage, audit_logger):
        self.storage = storage
        self.audit = audit_logger
    
    @staticmethod
    def compute_context_hash(context: dict) -> str:
        canonical = json.dumps(context, sort_keys=True)
        return hashlib.sha256(canonical.encode()).hexdigest()
    
    def lookup(self, tenant_id: str, context: dict, decision_type: str) -> Optional[dict]:
        h = self.compute_context_hash(context)
        row = self.storage.execute_fetchone(
            """SELECT * FROM decision_cache 
               WHERE tenant_id = ? AND context_hash = ? AND decision_type = ?
                 AND (expires_at IS NULL OR expires_at > datetime('now'))""",
            (tenant_id, h, decision_type)
        )
        if row:
            # Log a 'decision_reused' event
            self.audit.log(
                event_type='decision_reused',
                actor='orchestrator',
                action=f"Reused cached decision: {decision_type}",
                details=json.dumps({'decision_id': row['decision_id'], 'context_hash': h})
            )
            # Increment counters in cache (denormalized)
            self.storage.execute(
                """UPDATE decision_cache 
                   SET reuse_count = reuse_count + 1, last_reused_at = datetime('now')
                   WHERE decision_id = ?""",
                (row['decision_id'],)
            )
            return dict(row)
        return None
    
    def store(self, tenant_id: str, context: dict, decision_type: str, 
              decision_value: dict, scope_level: int, ttl_hours: Optional[int] = None,
              **metadata) -> str:
        h = self.compute_context_hash(context)
        decision_id = str(ULID())
        expires_at = None
        if ttl_hours:
            expires_at = (datetime.utcnow() + timedelta(hours=ttl_hours)).isoformat()
        
        # Log the decision_made event (source of truth)
        event_id = self.audit.log(
            event_type='decision_made',
            actor='orchestrator',
            action=f"Cached decision: {decision_type}",
            scope_level=scope_level,
            details=json.dumps({
                'decision_id': decision_id,
                'context_hash': h,
                'decision_type': decision_type,
                'decision_value': decision_value,
                'expires_at': expires_at,
                **metadata,
            }),
        )
        
        # Update materialized view
        self.storage.execute(
            """INSERT INTO decision_cache (
                decision_id, tenant_id, context_hash, decision_type, decision_value,
                scope_level, trust_score, created_at, expires_at, metadata
            ) VALUES (?, ?, ?, ?, ?, ?, ?, datetime('now'), ?, ?)""",
            (decision_id, tenant_id, h, decision_type, json.dumps(decision_value),
             scope_level, 0.5, expires_at, json.dumps(metadata))
```

```python
# orchestrator/routing_rules.py
class Router:
    def __init__(self, decision_cache, classifier, boundary_enforcer, audit):
        self.cache = decision_cache
        self.classifier = classifier
        self.boundary = boundary_enforcer
        self.audit = audit
    
    def route(self, tenant_id: str, action: str, context: dict, data: any) -> str:
        # 1. Check decision cache
        cached = self.cache.lookup(tenant_id, context, 'routing')
        if cached:
            return cached['routed_to']
        
        # 2. Classify
        complexity = self.classifier.assess_complexity(action, context)
        sensitivity = self.classifier.assess_sensitivity(data)
        
        # 3. Apply compliance boundary
        if not self.boundary.can_route_to(sensitivity, 'claude-api'):
            decision = 'ollama-local'
        elif complexity <= 3:
            decision = 'ollama-local'
        else:
            decision = 'claude-api'
        
        # 4. Cache the decision
        self.cache.store(
            tenant_id=tenant_id,
            context=context,
            decision_type='routing',
            decision_value={'routed_to': decision},
            scope_level=1,
            ttl_hours=24,
            complexity=complexity,
            sensitivity_level=sensitivity,
        )
        
        # 5. Audit log
        self.audit.log(
            event_type='routing_decision',
            actor='orchestrator',
            action=f"Routed '{action}' to {decision}",
            sensitivity_level=sensitivity,
            agent_routed_to=decision,
            details=json.dumps({'complexity': complexity, 'context_hash': self.cache.compute_context_hash(context)}),
        )
        
        return decision
```

### 13.4 Cloudflare Worker (Sync Endpoint)

```typescript
// worker/src/index.ts
export interface Env {
  AUDIT_DB: D1Database;
  CF_ACCESS_AUD: string;  // Cloudflare Access application AUD tag
}

interface SyncBatchPayload {
  machine_id: string;
  tenant_id: string;
  events: AuditEvent[];
}

interface SyncResponse {
  [event_id: string]: 'success' | 'duplicate_skipped' | string;  // error message
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    // 1. Validate Cloudflare Access JWT
    const jwt = request.headers.get('Cf-Access-Jwt-Assertion');
    if (!jwt) return new Response('Unauthorized', { status: 401 });
    
    const tokenIsValid = await validateAccessJwt(jwt, env.CF_ACCESS_AUD);
    if (!tokenIsValid) return new Response('Forbidden', { status: 403 });
    
    if (request.method !== 'POST') {
      return new Response('Method not allowed', { status: 405 });
    }
    
    // 2. Parse batch
    const payload: SyncBatchPayload = await request.json();
    const response: SyncResponse = {};
    
    // 3. Insert each event (D1 has UNIQUE on event_id; duplicates ignored)
    for (const event of payload.events) {
```

## 15. Compliance Considerations (HIPAA / Propel)

The `propel` tenant has `phi_allowed: true` in its tenant metadata. Special handling:

### 15.1 Routing Rules for `propel` Tenant

| Sensitivity | Allowed agents |
|-------------|----------------|
| `public` | Ollama, Claude, any agent |
| `internal` | Ollama, Claude (with PII redaction) |
| `confidential` | Ollama only |
| `sensitive_phi` | **Ollama only — Claude API explicitly blocked** |

### 15.2 Boundary Enforcement

`compliance_boundary.py` raises `BoundaryViolationError` if PHI data is routed to a non-local agent. This:
1. Aborts the subtask
2. Logs a `boundary_enforced` event
3. Notifies the user

### 15.3 HIPAA Evidence Trail

For an audit (e.g., from a Barclays-style third-party risk assessment), Propel can produce:

- **Audit log export** for any date range
- **Hash-chain verification** proving log integrity
- **PHI confinement report** showing 100% of `sensitive_phi` events routed to `ollama-local`
- **Boundary enforcement log** showing any attempted (and blocked) violations
- **Machine identity** showing which physical devices handled PHI

### 15.4 Retention

`propel` tenant: 2,555 days (7 years, HIPAA standard). Configured in `tenants.metadata`.

Older events archived to compressed Parquet files (out of scope for this ADR; see ADR-003 when written).

### 15.5 Cloudflare Considerations

- D1 stores **sanitized only** for Propel — no raw PHI ever leaves the local machine
- Cloudflare's BAA does not need to cover the agent's D1 usage (no PHI in D1)
- Local SQLite database should reside on encrypted disk (FileVault on macOS; BitLocker on Windows)
- Backup of local SQLite (if any) must follow Propel's data handling policy

## Notes & Open Questions

- **Performance**: Initial estimate is ~10 KB per event in SQLite. At 1,000 events/day, that's ~3.6 MB/year — D1 free tier (5 GB) supports years of usage.
- **Sync queue size monitoring**: Add an alert if unsynced events > 10,000 (could indicate network issues).
- **Machine ID rotation**: Currently UUIDs persist forever. If a machine is reimaged, treat it as a new machine. (Open: is there value in machine "fingerprinting" beyond a stored UUID?)
- **Multi-user on one machine**: Current design assumes one user per machine. If Sam's wife or another user runs the agent, add a `user_id` column. (Out of scope for v1.)
- **Encryption at rest**: Local SQLite relies on FileVault/BitLocker. D1 is encrypted by Cloudflare. (Open: column-level encryption for confidential events?)
- **Dashboard auth**: Cloudflare Access handles login. Tenant filtering done in Worker via JWT claims. (To be detailed in ADR-002.)
